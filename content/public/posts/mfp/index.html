Anatomy of DynamoDB table #

Table is the core of DynamoDB. DynamoDB Table is a very large, schemaless, and non-relational database table that consisted from key-value pairs.

  • DynamoDB tables stores zero or more items, which is a group of attributes that is uniquely identifiable, and we can store infinite numbers of items in tables.
  • Each table composed from:
    • Item: Groups of attributes that is uniquely identifiable.
      • Primary Key: Unique identifier for each item in the table. No two item can have same key. Two different of supported PK are:
        • Simple (partition key): only contains single partition key that ensures each item can be identified.
        • Composite (partition and sort key): contains two types of key that makes the item can be identified as well as sorted for better grouping and sorting.
      • An item can have one or more attributes. Attribute is data field within an item (similar to column) that represents specific piece of data. As DynamoDB is a NoSQL database, an item can have non, all mixture or different attributes.
      • Maximum size of an item is 400KB.
    • Capacity: amount of read and write capacity units a table has. Capacity can be grouped into provisioned and on-demand.
